An NLP-Based Prototype for Anomaly Detection in Cloud-Generated and Benchmark Cybersecurity Logs

Cloud servers, web applications, logins, and backend services are used daily in the modern organization. Such systems generate log data at a volume i.e. login attempts, failed access, service activity, system errors, and process events. They are also significant in cybersecurity as the logs are used to trace the activities occurring within the system and may reveal indicators of suspicious or dangerous activity. 
The point is that people have excessive log data that requires checking. Security administrators can not individually go through all the records, particularly when the normal operations and suspicious activity co-occur within a record. The larger and more cloud services are used by organizations, the bigger and more difficult to manage the amount of data is. 
It is possible that the cyber threats will occur in various manners. An instance is to have several unsuccessful logins, which could indicate a brute force attack. Patterns of strange events can display suspicious activity. A security problem can also be indicated by unexpected behavior of the server or application. Bearing in mind that such logs can be text-intensive and they might not be entirely structured, it might not be sufficient to do simple checks using keywords or rather give fixed rules. 
The usefulness of this project is that it addresses a real issue in contemporary organizations, which is the necessity to process large volumes of cybersecurity logs in an efficient way and enable the administrators to detect dangerous activity sooner. A prototype based on NLP is appropriate since it takes logs in the form of text which can be cleansed, tokenised and analysed to identify patterns. This endows the project with practical significance and definite scholastic importance.
